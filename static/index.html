About

I am an undergraduate computer science major at UCSD. Much of my time is spent writing various mods, plugins, and scripts, which I like to call "derpveloping" -- meaning poke the code enough until it compiles and does something cool, being able to apply the fundamentals I learned in school while learning new tricks and particular features of a language or a library along the way.

I particularly enjoy the logical aspect of programming, formulating a working algorithm that performs exactly what it's intended to do. To me, the most entertaining coding product is of course video games. I may not be a hardcore gamer who plays competitively, but it just floats my boat being able to create a game that other people will enjoy, or making modifications to a game that improve the experience.

  • Miscellaneous

    The possibilities in Minecraft are endless, and that's what I enjoy about it most. On the coding perspective, if I'm out of mod ideas, there are plenty of plugins for the server I could work on, or write various parsers that can comb through logs or provide statistics more easily. Or, if I don't feel like coding, I can build some dragons (actually playing the game, imagine that!), get myself a headache moderating for the server, or go kill some of those spammy, bothersome players in PvP... the list goes on. But here are some of the tangible works I can show.

  • Mods

    Almost every popular game has various player-created modifications that somehow change the game, for better or for worse. For Minecraft, I focus on using the LiteLoader API to create mods, granting the ability to make client-side changes without need for changing the internal game mechanics. All source code for my mods can be found on my Github, but those aren't necessarily very self-explanatory, so here's a nice showcase of all of them.
